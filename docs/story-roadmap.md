# evollard 스토리 로드맵

## 한 문장

13개 문제를 모두 풀면 `evollard`라는 가짜 미술품 판별 서비스가 완성되고, 최종 결과로 근거 기반 유료 감정 리포트가 발행된다.

## 서비스 흐름

```text
고객 감정 신청
  -> 작품/작가/소장 이력 저장
  -> 분석 Agent 실행
  -> 위작 위험도와 근거 생성
  -> 운영자 검수
  -> 유료 리포트 발행
  -> 매출/비용 기록
  -> 장애/운영 로그 보존
  -> GitHub 협업과 AI 자동화로 개선
```

## 장별 구성

### 1장. 돈과 기록

- 문제 11: 파일 기반 가계부
- evollard의 리포트 매출, 분석 비용, 클라우드 비용, 환불 내역을 파일로 기록한다.

### 2장. 감정 데이터의 뼈대

- 문제 5: SQL 데이터베이스
- 고객, 작가, 작품, 감정 요청, 유료 리포트 테이블을 설계한다.

### 3장. 감정 요청 백엔드

- 문제 4: FastAPI CRUD
- 감정 요청을 등록, 조회, 수정, 삭제하는 SSR 웹 서비스를 만든다.

### 4장. 보호된 리포트 서비스

- 문제 3: 인증과 연관관계
- 로그인한 고객만 리포트를 볼 수 있고, 운영자는 감정 상태를 변경한다.

### 5장. 고객/운영자 화면

- 문제 6: React SPA
- 문제 7: HTML/CSS/JS 포트폴리오
- 고객은 감정 진행 상태를 보고, 외부 방문자는 evollard의 신뢰 근거를 확인한다.

### 6장. 실제 서비스 운영

- 문제 2: AWS 인프라
- 문제 13: 시스템 관제 자동화
- 문제 12: 장애 트러블슈팅
- EC2에 배포하고, Agent 프로세스와 리포트 서버를 관제하며, OOM/CPU/Deadlock을 이슈로 남긴다.

### 7장. 내부 원리

- 문제 8: Mini Git
- 문제 9: Mini Redis
- 감정 리포트 변경 이력과 분석 결과 캐시 구조를 직접 구현하며 원리를 이해한다.

### 8장. 팀 개발 완성

- 문제 10: Git 협업
- 문제 1: AI Git 커밋/PR 생성기
- 팀원들이 Issue/PR/Review로 개발하고, AI가 변경사항을 커밋/PR 문서로 정리한다.
